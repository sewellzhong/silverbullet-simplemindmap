# SilverBullet SimpleMindMap Plug

Readonly inline rendering for Obsidian Simple Mind Map pages in SilverBullet.

## Install

Use `Library: Install` in SilverBullet with this URI:

```text
https://github.com/sewellzhong/silverbullet-simplemindmap/blob/main/PLUG.md
```

Then run `Plugs: Update`.

If you prefer direct plug installation through the SilverBullet `CONFIG` page, use this exact value:

```lua
config.set {
  plugs = {
    "github:sewellzhong/silverbullet-simplemindmap/simplemindmap.plug.js"
  }
}
```

Do not keep an empty string or placeholder such as `<your-user>` in `plugs`; the plug manager will report `Could not resolve plug uri`.

## Usage

- Open a page generated by Obsidian Simple mind map.
- The `metadata` block is rendered as a readonly mind map when it contains valid Obsidian Simple Mind Map data.
- The plug tries to hide Obsidian's raw `metadata/svgdata/linkdata/textdata` sections in SilverBullet without changing the note file.
- No modal or popup preview is opened.

This first version does not edit or save mind maps from SilverBullet. Use Obsidian Simple mind map as the editor and SilverBullet as the viewer.

## Config

```lua
config.set {
  simplemindmap = {
    height = 720,
    fit = true,
    minScale = 1.35,
    hideSourceSections = true
  }
}
```

`hideSourceSections` depends on SilverBullet's current page DOM. If a future SilverBullet release changes that structure, the map will still render but the raw Obsidian sections may become visible again until the plug is updated.

## Build

```bash
bun install
bun run verify
```

The compiled output is `simplemindmap.plug.js`.
